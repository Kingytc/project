Audit trial report
Audit trail is the ultimate security aspect in LAPS. This feature enables the administrator to view modifications / deletions carried out by every LAPS user on loan applications. In other words fixing responsibility of action for an action becomes possible and mishandling is reduced to a considerable extent.

    How to view the audit trial report
    Path - Setup ->Audit trial report
    A search page is displayed
    Enter the specific location of the LAPS user if you have the information
    Alternatively just enter the module and the time period within during which you wish see the audit trial report
    Date from - To has to be entered in the format dd/mm/yyyy
    You can see all LAPS users and the applications they have modified/ deleted through this report
    Or
    You can look for all borrower names and the applications they have submitted to the bank. Audit trial report will show if there are any changes in any of the application
    Enter details and click “Search”

    Click the LAPS user name/ borrower as the case may be
    If you have carried out a “Borrower search”, then a page with all available applications to which that borrower is a party is displayed:

    Select the “Short code”
    A screen with the following information (fields in this screen will be the same for both “User search” and “borrower search” but borrower search will only have applications specific to that borrower) will be displayed:
    Page name - Since the access is only to the administrator the JSP page name is only displayed
    User ID/ applicant name - LAPS user ID or the applicant name appears depending on your search criteria
    IP address (PC) - The IP address of the computer from which the action has been carried out is displayed
    Application No
    Action date - Date and time of action is displayed
    Data - All fields and the data in them is displayed. This can be compared to the original application form or with consequent record in this report itself if any further modifications have been carried out
    Action type (update / delete)
