Action

Click "Action" sub-tab
When you Click "Action" a new sub-tab called "remarks" appears next to the Action sub-tab

Action sub-tab is where you can carry out workflow actions, i.e., send the loan application from one person to another. A pre-defined path for the loan application travel will be displayed here; you will have to follow that path only. On having the necessary authority, you will be able to skip some Flow points in the approval/ rejection process.

Current flow point: Current flow point of the application is displayed here.
Received On: Date, on which the application reaches the user, is displayed.
Due On: The date and hour before which the action should be completed. This date is calculated based on the duration fixed for each action under ‘Setup --> Products --> flow point’.

Following details are displayed in tabular form:

Flow points: Flow point number through which the application passes through is displayed.
Done: Initial value in this field will be zero. On completion of the workflow process by the current user, it is changed to 1 automatically.
Minimum Class: A user who belongs to a "class" that is equal to or above the "Maximum class" specified can perform the specified action.
Actions: Description of the flow point as pre-defined by the system administrator is displayed here. There are In-process, Post-approval and post-rejection actions. Actions that are required to carried out when the loan processing is in progress is referred to as, "In-process" actions
Send: To send an application, these are the steps:
· Select the round radio button corresponding to the action.
· Select organization: Organization of the recipient to whom the current application should be sent can be selected
· Click "?", a list of all zones, regions and branches is displayed
· Select the location where you want to send the loan application. For instance, for a loan amount of Rs 5 lakhs, you may select your branch on the other hand, if the loan amount is Rs 25 lakhs then you may choose your Regional office

Select Recipient: The logged in user can select the recipient to whom the application should be forwarded
Click ‘?’, Recipients who satisfy the maximum class criteria in the organization selected are alone populated as a list of recipients in a popup window.
Select the desired recipient
Click "OK" button.
The name of the selected recipient gets displayed in this field and ‘Send’ button is enabled.
Click "Send button"

The next user can repeat the same process if he needs to send the application to the next
Flow point for another action

Approve or Reject an Application:

If all the actions have been completed and if the user has rights to "Approve" or "Reject" the loan application then, the ‘Approve’ and ‘Reject’ buttons are enabled

A user will be authorized to approve or reject an application, if the user’s class is equal to or more than the minimum class value specified in the product definition.

Approve - An authorized user can approve a loan by Clicking Approve button.
A user can then send the application to the next recipient for post-approval processing,
The minute, you click, "Approve" and the ‘Send’ button, Post-approval actions will be listed along with existing actions.

Reject - An authorized user can reject a loan by Clicking Reject button.
Once the loan is rejected, the user can send the application to the next recipient for post-rejection processing by selecting the next recipient and by Clicking ‘Send’ button. Post rejection actions will be listed under the action column along with the In -process actions.

On Click of ‘Send’ button, ‘Do you want to enter any remarks?’ message is displayed in an alert box.
Type remarks, if any
Click ‘OK’ in the alert box,

Undo - After processing a particular flow point, if the logged in user wants to reverse the last performed action, Click Undo button. This will return the application to pre-approval or pre-reject status. The system will display the In-process flow points.

Save - After completion of all flow points and on approval/ rejection of the application, Click ‘Save’ button, the application is stored and it cannot be modified later. The status of the form is changed to approved/ processed or rejected/processed depending upon the option specified.

Close - If the application form has been successfully processed, through all the flow points, Click Close button to close the application form. The status of the form is changed to approved/ Closed or rejected/ Closed, depending upon the action specified.

If the required action is "Approval" or "Rejection" and if you satisfy the "Minimum class" criteria for approval/ rejection, then both the "Approve & "Reject" buttons are enabled
Click one of them
Click "Send" to send it to the approving authority
Workflow comments screen as shown in is displayed.
